为某个特性或某个任务新建个branch吧，以免影响master的稳定性

嗯，master上好看多了 :)

不必担心 newBr 向 master 合两次甚至多次时产生冲突，git merge会自动记住你上次 merge 的节点，进而避开再次 merge。如果不想看到 newBr 上的 commit log，没啥意义，留着干啥，那就删掉吧：
git merge --squash newBr
